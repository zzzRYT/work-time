# S-03. 휴가 신청하기

> **탭**: Dashboard
> **관련 스펙**: REQ-1.4.1 ~ REQ-1.4.7
> **상태**: 🔴 미구현

## 유저 스토리

**AS** 스터디원
**I WANT TO** 2시간 단위로 휴가를 신청
**SO THAT** 불참 사유를 기록하고 출석률에 반영할 수 있다

```
→ [2h 선택 + 확인] → [출석 카드에 "휴가 2h" 표시] → [체크인 여전히 가능]
```

## 데이터 흐름

```
[휴가 확인] → useVacation(memberId, date, hours) → DailyVacation 생성
    → @@unique(memberId, date) 제약으로 중복 방지
    → hours == 8 → 상태 "VACATION"
    → hours < 8  → 기존 상태 유지, 휴가 정보만 추가 표시

[캘린더 반영] → calendar query → CalendarDay.status = VACATION (8h일 때)
[히스토리 반영] → dayDetail query → vacationHours 필드로 표시
[통계 반영] → monthlySummary → vacationDays 카운트 증가
```

## 서브 이슈

### S-03-1. 휴가 사용 버튼

> **우선순위**: 높음 | **상태**: 🔴 미구현

출석 카드 아래에 "휴가 사용" 보조 버튼을 배치한다.

**동작:**
| 상태 | 버튼 표시 | 스타일 |
|------|---------|-------|
| 미출석 | "🏖️ 휴가 사용" | 아웃라인 스타일, 체크인 버튼 아래 |
| 공부 중 | "🏖️ 휴가 사용" | 아웃라인 스타일, 체크아웃 버튼 아래 |
| 완료 | "🏖️ 휴가 사용" | 아웃라인 스타일, 다시 체크인 버튼 아래 |
| 이미 휴가 사용 | 비활성화 (회색) | disabled 상태 (S-03-6) |
| 전일 휴가 (8h) | 숨김 | — |

**구현 포인트:**
- 메인 액션 버튼(체크인/아웃) 아래에 보조 버튼으로 배치
- 탭 시 바텀시트 open (S-03-2)

### S-03-2. 바텀시트 시간 선택

> **우선순위**: 높음 | **상태**: 🔴 미구현

휴가 시간을 2h/4h/6h/8h 중 선택할 수 있는 바텀시트를 표시한다.

**동작:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|------------|-----------|
| 1 | 바텀시트 열림 | 시간 선택 칩 4개 표시 (선택 없음 초기 상태) |
| 2 | 시간 칩 탭 (e.g. 4h) | 해당 칩 하이라이트, 나머지 비선택 |
| 3 | "확인" 버튼 탭 | mutation 호출 → 바텀시트 닫힘 |
| 4 | 바텀시트 바깥 탭 또는 스와이프 다운 | 선택 취소, 바텀시트 닫힘 |

**UI:**
```
┌─────────────────────────────────┐
│  휴가 시간 선택                    │
│                                 │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│  │  2h  │ │ [4h] │ │  6h  │ │  8h  │  ← 선택 칩
│  └──────┘ └──────┘ └──────┘ └──────┘
│                                 │
│  ┌─────────────────────────┐    │
│  │         확인              │    │  ← 시간 미선택 시 비활성화
│  └─────────────────────────┘    │
└─────────────────────────────────┘
```

**구현 포인트:**
- `@gorhom/bottom-sheet` 라이브러리 활용
- 확인 버튼은 시간 미선택 시 disabled

### S-03-3. 8h 전일 휴가 경고

> **우선순위**: 중간 | **상태**: 🔴 미구현

바텀시트에서 8h를 선택하면 전일 휴가 안내 문구를 표시한다.

**동작:**
| 선택 | 안내 문구 | 표시 위치 |
|------|---------|---------|
| 2h / 4h / 6h | 없음 | — |
| 8h | "⚠️ 전일 휴가로 처리되며, 오늘 체크인이 불가합니다" | 칩 아래, 확인 버튼 위 |

**구현 포인트:**
- 선택된 시간이 8일 때 조건부 경고 텍스트 렌더링
- 노란/주황 배경 + 경고 아이콘

### S-03-4. 날짜 선택 (다중 선택)

> **우선순위**: 높음 | **상태**: 🔴 미구현

오늘뿐 아니라 다른 날짜도 선택 가능하며, 한 번에 여러 날짜를 선택하여 휴가를 일괄 등록할 수 있다.

**동작:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|------------|-----------|
| 1 | 바텀시트 열림 | 기본 선택: 오늘 날짜 |
| 2 | 날짜 영역 탭 | 미니 캘린더 또는 날짜 피커 표시 |
| 3 | 날짜 탭 (복수) | 선택된 날짜들 하이라이트 + 선택 카운트 표시 |
| 4 | 시간 선택 + 확인 | 선택된 모든 날짜에 동일 시간 휴가 등록 |

**UI:**
```
┌─────────────────────────────────┐
│  휴가 신청                        │
│                                 │
│  날짜 선택                        │
│  ┌─────────────────────────┐    │
│  │  3/11(화)  3/12(수)  3/13(목) │  ← 선택된 날짜 칩들
│  │  [+ 날짜 추가]              │    │  ← 탭하면 캘린더 피커
│  └─────────────────────────┘    │
│                                 │
│  휴가 시간 선택                    │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│  │  2h  │ │ [4h] │ │  6h  │ │  8h  │
│  └──────┘ └──────┘ └──────┘ └──────┘
│                                 │
│  ┌─────────────────────────┐    │
│  │     확인 (3일 등록)        │    │  ← 선택 날짜 수 표시
│  └─────────────────────────┘    │
└─────────────────────────────────┘
```

**데이터 흐름:**
```
[확인 탭] → 선택된 날짜별로 useVacation mutation 순차/병렬 호출
    → useVacation(memberId, "2026-03-11", 4)
    → useVacation(memberId, "2026-03-12", 4)
    → useVacation(memberId, "2026-03-13", 4)
    → 각 날짜별 DailyVacation 레코드 생성 (DB 저장)
```

**서버 변경 필요:**
- 기존 `useVacation`은 단일 날짜 → 배치 mutation 추가 또는 클라이언트에서 다건 호출
- `@@unique(memberId, date)` 제약으로 같은 날 중복은 자동 방지

**구현 포인트:**
- 바텀시트 내에 미니 캘린더 컴포넌트 (다중 선택 모드)
- 선택된 날짜를 칩 형태로 표시 + 제거 가능
- 확인 버튼에 선택 건수 표시

### S-03-5. 출석 카드 휴가 정보 표시

> **우선순위**: 높음 | **상태**: 🔴 미구현

휴가 사용 후 출석 카드에 사용한 휴가 시간을 표시한다.

**동작:**
| 상태 | 표시 내용 | UI 위치 |
|------|---------|---------|
| 반차 (2h/4h/6h) | "🏖️ 휴가 4h 사용" | 출석 카드 상태 뱃지 아래 |
| 전일 (8h) | "🏖️ 휴가중" | 출석 카드 중앙 (S-03-6) |
| 휴가 없음 | 표시 없음 | — |

**구현 포인트:**
- `dayDetail` 또는 별도 query로 오늘의 `vacationHours` 조회
- `vacationHours > 0 && vacationHours < 8` → 반차 정보 표시
- `vacationHours === 8` → S-03-6으로 위임

### S-03-6. 전일 휴가 시 "휴가중" 표시

|------|------|

| 체크인 버튼 | 숨김 |
| 휴가 사용 버튼 | 숨김 |

**UI:**
```
┌───────────────────────────┐
│  상태: 🟡 휴가중             │
│                           │
│    🏖️ 오늘은 휴가입니다       │  ← 타이머 대신 안내
```

### S-03-7. 중복 신청 방지

> **우선순위**: 중간 | **상태**: 🔴 미구현

이미 오늘 휴가를 사용한 경우 추가 신청을 방지한다.

**동작:**
| 상황 | 시스템 반응 | UI |
|------|-----------|-----|
| 오늘 휴가 미사용 | 버튼 활성화, 정상 동작 | 기본 스타일 |
| 오늘 이미 휴가 사용 | 버튼 비활성화 (회색) | disabled + "이미 사용됨" 텍스트 |
| 다중 날짜 선택 시 이미 사용된 날짜 | 해당 날짜 선택 불가 | 캘린더에서 회색 처리 + 안내 |

**구현 포인트:**
- 오늘의 `vacationHours` 조회 → 0이 아니면 버튼 disabled
- 다중 날짜 선택 시 이미 DailyVacation이 있는 날짜는 선택 불가 표시
- 서버 측: `@@unique(memberId, date)` 제약이 최종 방어선

### S-03-8. 캘린더 휴가 도트

> **우선순위**: 중간 | **상태**: 🔴 미구현

**동작:**

|----------|------|------|
| 전일 휴가 (8h) | ● | 🟡 노랑 |

### S-03-10. 월간 통계 휴가 일수 반영

> **우선순위**: 중간 | **상태**: 🔴 미구현
> **연관**: S-07 (월간 통계 확인)

월간 요약에서 해당 월의 휴가 사용 일수를 카운트하여 표시한다.

**동작:**
- `monthlySummary.vacationDays`에 DailyVacation 레코드 수 반영
- 반차(2h~6h)도 1일로 카운트? 또는 시간 합산으로 환산? → **1건 = 1일로 카운트**

**구현 포인트:**
- 서버 `monthlySummary` resolver에서 vacationDays 계산 (이미 구현됨)
