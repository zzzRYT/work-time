# S-08. 멤버 목록에서 오늘의 출석 현황 보기

> **탭**: Members
> **관련 스펙**: REQ-1.1.1 ~ REQ-1.2.8 (Members)
> **상태**: 🟡 부분 구현

---

## 유저 스토리

**AS** 스터디원
**I WANT TO** 스터디원 전체의 오늘 출석 현황을 한 화면에서 확인
**SO THAT** 누가 출석했고, 지각했고, 공부 중인지 파악할 수 있다

---

## 플로우

### 플로우 1: 출석 요약 + 멤버 목록 조회

```
[멤버 탭 진입] → [요약 카드: 전체 5명 / 출석 3명 / 공부 중 2명 / 지각 1명]
→ [멤버 목록: 공부 중인 멤버 상단 정렬]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 멤버 탭 진입 | `todayAttendanceSummary` + `members` query | — |
| 2 | — | 요약 카드 (4개 카운트) 표시 | 화면 상단 |
| 3 | — | 멤버 목록 (공부 중 → 나머지 순 정렬) | 요약 카드 아래 |
| 4 | — | 각 멤버: 아바타 + 이름 + 상태 + 오늘 공부 시간 | 멤버 행 |

### 플로우 2: 멤버 상태별 표시

| 멤버 상태 | 표시 요소 | UI 위치 |
|----------|----------|---------|
| 공부 중 | 🟢 초록 인디케이터 + "공부 중" + 경과 시간 | 멤버 행 우측 |
| 완료 | 오늘 총 공부 시간 | 멤버 행 우측 |
| 지각 | 🔴 "지각" 뱃지 + 공부 시간 | 이름 옆 + 멤버 행 우측 |
| 휴가 | 🟡 "휴가" 뱃지 | 이름 옆 |
| 미출석 | 회색 텍스트 "미출석" | 멤버 행 우측 |

## 데이터 흐름

```
[화면 진입]
    → todayAttendanceSummary → { total, attended, studying, late }
    → members → [{ id, name, displayName, color, currentStatus, todayStudyMinutes }]

[정렬 로직]
    1. currentStatus == STUDYING → 상단
    2. currentStatus == COMPLETED / LATE → 중간
    3. currentStatus == VACATION → 하단
    4. currentStatus == NOT_ATTENDED → 최하단

[polling 30s] → 실시간 상태 갱신
```

## 서브 이슈

### S-08-1. 상태별 뱃지/인디케이터

> **우선순위**: 높음 | **상태**: 🔴 미구현

각 멤버의 현재 출석 상태에 따라 뱃지와 인디케이터를 표시한다.

**상태별 표시:**
| 상태 | 뱃지 | 인디케이터 | 우측 정보 |
|------|------|----------|---------|
| 공부 중 | — | 🟢 초록 점 (이름 옆) | 경과 시간 "1h 45m" |
| 완료 | — | — | 오늘 총 공부 시간 "3h 00m" |
| 지각 (공부 중) | 🟡 "지각" (노랑) | 🟢 초록 점 | 경과 시간 |
| 지각 (완료) | 🟡 "지각" (노랑) | — | 총 공부 시간 |
| 휴가 | 🔵 "휴가" (파랑) | — | — |
| 미출석 | ⬜ "미출석" (회색) | — | — |

**뱃지 스타일:**
- 이름 우측에 작은 뱃지 태그
- S-05 색상 체계와 통일 (지각=노랑, 휴가=파랑)

**미출석 처리:**
- 모든 미출석 멤버에 회색 "미출석" 뱃지를 달아서 상태가 명확히 보이도록 함
- 이름 + 아바타도 약간 투명하게 (opacity-60)

**구현 포인트:**
- `currentStatus` 기반 조건부 렌더링
- StatusBadge 공유 컴포넌트 활용 (variant: studying / late / vacation / absent)
- 인디케이터: 이름 왼쪽 작은 원형 (8px)

### S-08-2. 멤버 행 탭 → 출석 상세

> **우선순위**: 중간 | **상태**: 🔴 미구현
> **연관**: S-04-3

멤버 행을 탭하면 해당 멤버의 오늘 출석 상세를 바텀시트로 표시한다.

**동작:**
| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 멤버 행 탭 | `dayDetail(memberId, today)` query | 멤버 행 |
| 2 | — | 바텀시트 열림: 해당 멤버의 세션 목록 | 바텀시트 |

**바텀시트 내용:**
```
┌─────────────────────────────────┐
│  🟣 홍길동의 오늘 기록              │
│                                 │
│  09:55 → 12:30  (2h 35m)       │
│  🟡지각 13:15 → 진행 중  (1h~)   │
│                                 │
│  오늘 총: 3시간 35분~              │
└─────────────────────────────────┘
```

**구현 포인트:**
- S-04-3과 동일한 바텀시트 컴포넌트 공유
- S-06 일별 상세 컴포넌트를 바텀시트 안에서 재사용

### S-08-3. Pull-to-refresh

> **우선순위**: 낮음 | **상태**: 🔴 미구현

멤버 목록을 아래로 당겨서 데이터를 새로고침한다.

**동작:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|------------|-----------|
| 1 | 목록 상단에서 아래로 당김 | 스피너 표시 |
| 2 | — | `todayAttendanceSummary` + `members` refetch |
| 3 | 응답 완료 | 스피너 숨김, 데이터 갱신 |

**구현 포인트:**
- FlatList/ScrollView의 `refreshControl` prop 활용
- `RefreshControl` 컴포넌트 + Apollo `refetch` 연결
