# S-10. 주간/월간 랭킹 확인하기

> **탭**: Members
> **관련 스펙**: REQ-1.4.1 ~ REQ-1.4.8 (Members)
> **상태**: 🟡 부분 구현

---

## 유저 스토리

**AS** 스터디원
**I WANT TO** 주간/월간 공부 시간 랭킹을 확인
**SO THAT** 다른 멤버와 비교하며 동기부여를 받을 수 있다

---

## 플로우

### 플로우 1: 랭킹 조회

```
[멤버 탭] → [스크롤 → 랭킹 섹션] → [이번 주 / 이번 달 탭 전환]
→ [멤버별 순위 + 공부 시간 + 출석일 + 지각 횟수]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 멤버 탭에서 스크롤 | 랭킹 섹션 노출 | 회비/지각비 아래 섹션 |
| 2 | — | 기본 "이번 주" 탭 선택 상태 | 탭 전환 바 |
| 3 | — | `memberRanking(WEEKLY)` query | — |
| 4 | — | 순위별 멤버 리스트 렌더링 | 랭킹 목록 |

### 플로우 2: 기간 전환

```
["이번 달" 탭 선택] → [월간 랭킹으로 전환]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | "이번 달" 탭 탭 | `memberRanking(MONTHLY)` query | 탭 전환 바 |
| 2 | — | 월간 랭킹 데이터로 갱신 | 랭킹 목록 |

## 데이터 흐름

```
memberRanking(period: WEEKLY | MONTHLY)
    → [{ member, totalStudyMinutes, attendanceDays, lateCount }]
    → 총 공부 시간 내림차순 정렬 (서버에서 정렬)

WEEKLY: 이번 주 월~일 기준
MONTHLY: 이번 달 1일~말일 기준
```

## 서브 이슈

### S-10-1. 랭킹 리스트 렌더링

> **우선순위**: 높음 | **상태**: 🔴 미구현

순위별 멤버 행을 리스트로 렌더링한다.

**멤버 행 구조:**
```
┌─────────────────────────────────┐
│ 🥇 🟣 홍길동                      │  ← 순위 아이콘 + 아바타 + 이름
│    32h 15m · 출석 5일 · 지각 0회   │  ← 통계 (서브 텍스트)
└─────────────────────────────────┘
```

**표시 항목:**
| 위치 | 데이터 | 포맷 |
|------|-------|------|
| 좌측 | 순위 아이콘/숫자 | 🥇🥈🥉 또는 "4" |
| 좌측 | 아바타 (프로필 색상) | 원형 |
| 좌측 | 이름 | displayName |
| 하단 | 총 공부 시간 | "Nh Mm" |
| 하단 | 출석 일수 | "출석 N일" |
| 하단 | 지각 횟수 | "지각 N회" |

**구현 포인트:**
- `memberRanking` query 결과를 FlatList로 렌더링
- 통계는 `·` 구분자로 한 줄 표시
- S-07-2 포맷 유틸 함수 재사용

---

### S-10-2. 주간/월간 탭 전환

> **우선순위**: 높음 | **상태**: 🔴 미구현

"이번 주" / "이번 달" 탭으로 랭킹 기간을 전환한다.

**동작:**
| 탭 | query 파라미터 | 기간 |
|----|-------------|------|
| 이번 주 (기본) | `WEEKLY` | 이번 주 월~일 |
| 이번 달 | `MONTHLY` | 이번 달 1일~말일 |

**UI:**
```
┌──────────┬──────────┐
│ [이번 주] │  이번 달  │  ← 선택 탭: 채움 배경 + 볼드
└──────────┴──────────┘
```

**스타일:**
- 선택된 탭: Primary 색상 배경 + 흰색 텍스트
- 비선택 탭: 투명 배경 + 회색 텍스트

**구현 포인트:**
- `period` 상태 (WEEKLY / MONTHLY) 관리
- 탭 변경 시 `memberRanking(period)` query variables 변경 → 자동 refetch

---

### S-10-3. 순위 메달 아이콘

> **우선순위**: 중간 | **상태**: 🔴 미구현

1~3등에 메달 아이콘, 4등 이하에 숫자를 표시한다.

**매핑:**
| 순위 | 표시 | 스타일 |
|------|------|-------|
| 1등 | 🥇 | 금색 배경 원 또는 이모지 |
| 2등 | 🥈 | 은색 배경 원 또는 이모지 |
| 3등 | 🥉 | 동색 배경 원 또는 이모지 |
| 4등~ | "4", "5"... | 회색 텍스트, 동일 크기 영역 |

**구현 포인트:**
- `rankToIcon(rank: number)` 유틸 함수
- 순위 영역 고정 너비 (32px)로 정렬 유지

---

### S-10-4. 내 순위 강조

> **우선순위**: 중간 | **상태**: 🔴 미구현

현재 선택된(로그인한) 멤버의 랭킹 행을 시각적으로 강조한다.

**동작:**
| 행 | 스타일 |
|----|-------|
| 내 행 | 배경 하이라이트 (Primary 연한 색 `bg-indigo-50`) + 좌측 보더 |
| 다른 멤버 행 | 기본 배경 |

**UI:**
```
│  🥈 🔵 김철수                     │  ← 일반
│┃ 🥉 🟢 이영희 (나)                │  ← 내 행: 좌측 보더 + 배경 + "(나)" 표시
│  4  🟠 박지민                     │  ← 일반
```

**구현 포인트:**
- `selectedMemberId === entry.member.id` 조건부 스타일
- "(나)" 텍스트 또는 작은 뱃지로 구분

---

### S-10-5. 빈 랭킹

> **우선순위**: 낮음 | **상태**: 🔴 미구현

해당 기간에 출석 기록이 없을 때 안내 문구를 표시한다.

**동작:**
| 조건 | 표시 |
|------|------|
| 랭킹 데이터 0건 | "이번 주 기록이 없습니다" / "이번 달 기록이 없습니다" |

**UI:**
```
┌──────────┬──────────┐
│ [이번 주] │  이번 달  │
└──────────┴──────────┘

    이번 주 기록이 없습니다     ← 회색 텍스트, 가운데 정렬
```

**구현 포인트:**
- `rankingData.length === 0` 조건부 빈 상태 렌더링
- 선택된 탭에 따라 "이번 주" / "이번 달" 문구 변경
