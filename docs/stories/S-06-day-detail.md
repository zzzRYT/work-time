# S-06. 일별 출석 상세 확인하기

> **탭**: History
> **관련 스펙**: REQ-1.2.1 ~ REQ-1.2.9 (History)
> **상태**: 🟡 부분 구현

---

## 유저 스토리

**AS** 스터디원
**I WANT TO** 특정 날짜의 출석 세션을 상세히 확인
**SO THAT** 그날 몇 번, 언제, 얼마나 공부했는지 알 수 있다

## 플로우

### 플로우 1: 출석 기록이 있는 날

```
[캘린더에서 날짜 선택] → [세션 목록 표시]
→ [체크인 시간 / 체크아웃 시간 / 공부 시간 확인] → [총 공부 시간 확인]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 캘린더에서 날짜 탭 | `dayDetail` query 호출 | — |
| 2 | — | 해당 날짜의 세션 목록 렌더링 | 캘린더 아래 "일별 상세" 섹션 |
| 3 | — | 각 세션: 체크인 시간 → 체크아웃 시간 (공부 시간) | 세션 행 |
| 4 | — | 지각 세션: 체크인 시간 옆에 "지각" 뱃지 | 세션 행 좌측 |
| 5 | — | 해당 날짜 총 공부 시간 표시 | 섹션 하단 요약 |

### 플로우 2: 진행 중인 세션 (오늘)

```
[오늘 날짜 선택] → [마지막 세션: 체크아웃 = "진행 중"]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 오늘 날짜 선택 | 세션 목록에 현재 세션 포함 | 세션 목록 |
| 2 | — | 체크아웃 시간 → "진행 중" 텍스트 표시 | 세션 행 체크아웃 위치 |

### 플로우 3: 휴가가 있는 날

```
[휴가 사용한 날짜 선택] → [세션 목록 + "휴가 4h" 표시]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 휴가 사용 날짜 선택 | 세션 목록 + 휴가 정보 표시 | 일별 상세 섹션 |
| 2 | — | "🏖️ 휴가 4h" 형태로 표시 | 세션 목록 상단 또는 별도 행 |

### 플로우 4: 기록이 없는 날

```
[출석 기록 없는 날짜 선택] → ["기록이 없습니다" 안내]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 빈 날짜 선택 | "기록이 없습니다" 빈 상태 표시 | 일별 상세 섹션 |

## 데이터 흐름

```
dayDetail(memberId, date)
    → sessions: [{ checkInTime, checkOutTime, isLate, durationMinutes }]
    → totalDurationMinutes: 전체 합산
    → vacationHours: 0 | 2 | 4 | 6 | 8

* checkOutTime == null → "진행 중" 표시 + 현재까지 경과 시간
* isLate == true → 해당 세션에 "지각" 뱃지
* vacationHours > 0 → 휴가 행 추가 표시
```

## 서브 이슈

### S-06-1. 세션 목록 렌더링

> **우선순위**: 높음 | **상태**: 🔴 미구현

선택된 날짜의 출석 세션을 체크인 → 체크아웃 (공부 시간) 형태로 리스트 렌더링한다.

**동작:**
| 요소 | 표시 형식 | 예시 |
|------|---------|------|
| 체크인 시간 | HH:MM | 09:55 |
| 화살표 | → | → |
| 체크아웃 시간 | HH:MM | 12:30 |
| 공부 시간 | (Nh Mm) | (2h 35m) |

**UI:**
```
┌─────────────────────────┐
│ 09:55 → 12:30  (2h 35m) │  ← 세션 행
├─────────────────────────┤
│ 13:00 → 17:00  (4h 0m)  │
└─────────────────────────┘
```

**구현 포인트:**
- `dayDetail.sessions` 배열을 FlatList 또는 map으로 렌더링
- 시간 포맷: ISO → KST 변환 후 HH:MM 표시
- 공부 시간: `durationMinutes` → "Nh Mm" 포맷

---

### S-06-2. 진행 중 세션 표시

> **우선순위**: 높음 | **상태**: 🔴 미구현

체크아웃이 아직 안 된 세션(오늘)의 체크아웃 위치에 "진행 중"을 표시한다.

**동작:**
| 조건 | 체크아웃 표시 | 공부 시간 표시 |
|------|------------|-------------|
| checkOutTime != null | HH:MM | (Nh Mm) |
| checkOutTime == null | "진행 중" (초록 텍스트) | (Nh Mm~) 틸드 포함 |

**UI:**
```
│ 09:55 → 진행 중  (1h 30m~) │  ← "진행 중" 초록 텍스트 + 틸드
```

**구현 포인트:**
- `checkOutTime === null` → "진행 중" 텍스트 + `text-green-500`
- 공부 시간: `Date.now() - checkInTime` 기준 계산 + "~" 접미사

---

### S-06-3. 지각 세션 뱃지

> **우선순위**: 중간 | **상태**: 🔴 미구현
> **연관**: S-02-5

지각 판정된 세션의 체크인 시간 앞에 "지각" 뱃지를 표시한다.

**동작:**
| 조건 | 표시 |
|------|------|
| isLate == false | `09:55 → 12:30  (2h 35m)` |
| isLate == true | `🟡지각 13:15 → 17:00  (3h 45m)` |

**뱃지 스타일:** 노란 배경 + 흰색 "지각" 텍스트 (S-05 색상 체계와 통일: 지각=노랑)

**구현 포인트:**
- `session.isLate` 기반 조건부 뱃지 렌더링
- StatusBadge 공유 컴포넌트 활용 (variant: "late")

---

### S-06-4. 총 공부 시간 합산

> **우선순위**: 높음 | **상태**: 🔴 미구현

세션 목록 하단에 해당 날짜의 모든 세션 공부 시간을 합산하여 표시한다.

**동작:**
| 세션 수 | 표시 |
|--------|------|
| 1개 | 총 공부 시간: 2시간 35분 |
| 복수 | 총 공부 시간: 6시간 20분 (세션 합산) |
| 0개 | 표시 안 함 (S-06-6 빈 상태) |
| 진행 중 포함 | 총 공부 시간: 4시간 5분~ (틸드 포함) |

**UI 위치:** 세션 목록 하단, 구분선 아래
```
│  총 공부 시간: 6시간 20분  │  ← 볼드 텍스트
```

**구현 포인트:**
- `dayDetail.totalDurationMinutes` 사용
- 진행 중 세션 포함 시 실시간 계산 + "~" 접미사

---

### S-06-5. 휴가 정보 표시

> **우선순위**: 중간 | **상태**: 🔴 미구현
> **연관**: S-03-9

세션 목록 상단에 해당 날짜의 휴가 사용 정보를 별도 행으로 표시한다.

**동작:**
| 조건 | 표시 |
|------|------|
| vacationHours == 0 | 표시 안 함 |
| vacationHours == 2~6 | "🏖️ 휴가 4h" (반차) |
| vacationHours == 8 | "🏖️ 휴가 8h (전일)" |

**UI 위치:** 세션 목록 상단 (세션 행들 위)
```
│  🏖️ 휴가 4h               │  ← 파란 배경 행
│                           │
│  14:00 → 18:00  (4h 0m)  │  ← 반차 후 출석 세션
```

**구현 포인트:**
- `dayDetail.vacationHours > 0` → 세션 목록 위에 휴가 행 렌더링
- 파란 배경 (S-05 색상 체계: 휴가=파랑)

---

### S-06-6. 빈 상태 UI

> **우선순위**: 중간 | **상태**: 🔴 미구현

출석 기록이 없는 날짜를 선택했을 때 안내 메시지를 표시한다.

**동작:**
| 조건 | 표시 |
|------|------|
| sessions.length == 0 && vacationHours == 0 | "기록이 없습니다" 안내 |

**UI:**
```
┌─────────────────────────────────┐
│  3월 15일 (토)                    │
│                                 │
│       기록이 없습니다              │  ← 회색 텍스트, 가운데 정렬
│                                 │
└─────────────────────────────────┘
```

**구현 포인트:**
- 세션 + 휴가 모두 없을 때 조건부 빈 상태 렌더링
- 주말/미래 날짜도 동일하게 "기록이 없습니다" 표시
