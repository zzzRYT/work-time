# S-13. 멤버 선택하기 (앱 진입)

> **탭**: 공통 (앱 전체)
> **관련 스펙**: 없음 (MVP 간소화 인증)
> **상태**: 🟡 부분 구현

---

## 유저 스토리

**AS** 스터디원
**I WANT TO** 앱 진입 시 내 프로필을 선택
**SO THAT** 나의 출석을 기록하고 내 기록을 조회할 수 있다

---

## 배경

MVP에서는 별도 인증(로그인) 없이, 멤버 목록에서 자신을 선택하는 방식으로 사용자를 식별한다.
선택한 멤버 ID는 클라이언트 상태(Zustand 또는 AsyncStorage)로 유지한다.

---

## 플로우

### 플로우 1: 최초 진입 (멤버 미선택)

```
[앱 실행] → [멤버 선택 화면] → [내 이름 탭]
→ [대시보드로 이동] → [선택한 멤버 기준으로 모든 데이터 표시]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 앱 실행 | 저장된 멤버 ID 확인 → 없음 | — |
| 2 | — | 멤버 선택 화면 표시 | 전체 화면 (모달 또는 별도 라우트) |
| 3 | — | 전체 멤버 목록 (아바타 + 이름) 표시 | 선택 화면 중앙 |
| 4 | 내 이름 탭 | 멤버 ID 저장 (로컬) | — |
| 5 | — | 대시보드로 자동 이동 | 탭 네비게이션 |

### 플로우 2: 재진입 (멤버 이미 선택)

```
[앱 실행] → [저장된 멤버 ID 있음] → [바로 대시보드 표시]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 앱 실행 | 저장된 멤버 ID 확인 → 있음 | — |
| 2 | — | 대시보드 바로 표시 | 탭 네비게이션 |

### 플로우 3: 멤버 변경

```
[설정/프로필 영역] → [멤버 변경] → [멤버 선택 화면] → [다른 멤버 선택]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 프로필/설정 영역 탭 | 멤버 변경 옵션 표시 | 대시보드 상단 또는 설정 |
| 2 | "멤버 변경" 탭 | 멤버 선택 화면 표시 | 전체 화면 |
| 3 | 다른 멤버 선택 | 멤버 ID 교체 저장 | — |
| 4 | — | 모든 화면 데이터 갱신 (새 멤버 기준) | 전체 탭 |

## 데이터 흐름

```
[앱 시작] → AsyncStorage/Zustand에서 selectedMemberId 로드
    → null → 멤버 선택 화면
    → 있음 → 대시보드 진입

[멤버 선택] → selectedMemberId 저장
    → 모든 query의 memberId 파라미터로 사용
    → Dashboard: activeSession(memberId), checkIn(memberId)
    → History: calendar(memberId), dayDetail(memberId)
    → Admin: role 확인 → 관리 탭 가시성

[멤버 변경] → selectedMemberId 교체
    → Apollo cache 초기화 또는 refetch
    → 모든 화면 데이터 새 멤버 기준으로 갱신
```

## 서브 이슈

### S-13-1. 멤버 선택 화면

> **우선순위**: 높음 | **상태**: 🔴 미구현

전체 멤버를 그리드 카드 형태로 표시하고, 탭하여 선택하는 별도 화면.

**UI:**
```
┌─────────────────────────────────┐
│                                 │
│        누구세요?                  │  ← 타이틀 (가운데, 큰 텍스트)
│                                 │
│  ┌──────────┐  ┌──────────┐     │
│  │   🟣     │  │   🔵     │     │
│  │  홍길동   │  │  김철수   │     │  ← 2열 그리드 카드
│  └──────────┘  └──────────┘     │
│  ┌──────────┐  ┌──────────┐     │
│  │   🟢     │  │   🟠     │     │
│  │  이영희   │  │  박지민   │     │
│  └──────────┘  └──────────┘     │
│  ┌──────────┐                   │
│  │   🔴     │                   │
│  │  최수정   │                   │
│  └──────────┘                   │
│                                 │
└─────────────────────────────────┘
```

**카드 구성:**
| 요소 | 설명 |
|------|------|
| 아바타 | 멤버 color 기반 원형 (크게, 48px+) |
| 이름 | displayName (아바타 아래) |
| 탭 피드백 | 탭 시 카드 살짝 눌림 효과 (scale 0.95) |

**동작:**
| 단계 | 행동 | 결과 |
|------|------|------|
| 1 | 멤버 카드 탭 | selectedMemberId 저장 (AsyncStorage + Zustand) |
| 2 | — | 대시보드로 자동 이동 |

**구현 포인트:**
- Expo Router 별도 라우트 (`/select-member` 또는 모달)
- `members` query로 목록 가져오기
- 2열 그리드: `flexWrap: wrap` 또는 `FlatList numColumns={2}`

### S-13-2. 앱 시작 라우팅 분기

> **우선순위**: 높음 | **상태**: 🔴 미구현

앱 시작 시 저장된 멤버 ID 유무에 따라 화면을 분기한다.

**분기 로직:**
```
앱 시작 → AsyncStorage에서 selectedMemberId 로드
    → null / undefined → 멤버 선택 화면으로 이동
    → 유효한 ID → 대시보드(탭 네비게이션)로 이동
```

**엣지 케이스:**
| 상황 | 처리 |
|------|------|
| 저장된 ID가 더 이상 유효하지 않음 (멤버 삭제 등) | 멤버 선택 화면으로 이동 + 저장값 초기화 |
| AsyncStorage 로딩 중 | 스플래시/로딩 화면 유지 |

**구현 포인트:**
- `app/_layout.tsx`에서 AsyncStorage 로드 → 라우팅 결정
- Expo Router `Redirect` 컴포넌트 또는 `router.replace()` 활용
- 로딩 중 빈 화면 방지: `expo-splash-screen` keep visible

### S-13-3. AsyncStorage 영속 저장

> **우선순위**: 높음 | **상태**: 🔴 미구현

선택한 멤버 ID를 AsyncStorage에 저장하여 앱 종료 후에도 유지한다.

**저장 규격:**
| 키 | 값 | 예시 |
|----|---|------|
| `@work-time/selected-member-id` | memberId string | `"clx1abc..."` |

**동작:**
| 시점 | 동작 |
|------|------|
| 멤버 선택 | `AsyncStorage.setItem(key, memberId)` |
| 앱 시작 | `AsyncStorage.getItem(key)` → Zustand store에 세팅 |
| 멤버 변경 | `AsyncStorage.setItem(key, newMemberId)` |

**Zustand store 연동:**
```typescript
// shared/store/member.ts
interface MemberStore {
  selectedMemberId: string | null
  setSelectedMemberId: (id: string) => void
  clearSelectedMemberId: () => void
  isLoaded: boolean  // AsyncStorage 로드 완료 여부
}
```

**구현 포인트:**
- `@react-native-async-storage/async-storage` 사용
- Zustand persist middleware 또는 수동 hydration
- `isLoaded` 플래그로 로딩 상태 관리 (S-13-2 분기에 활용)

### S-13-4. 멤버 변경 기능

> **우선순위**: 중간 | **상태**: 🔴 미구현

대시보드 상단에서 현재 선택된 멤버를 탭하면 멤버 변경이 가능하다.

**진입점:**
```
┌─────────────────────────────────┐
│  🟣 홍길동  ▼                    │  ← 탭하면 멤버 선택 화면으로 이동
│                                 │
│  (출석 카드 영역...)               │
└─────────────────────────────────┘
```

**동작:**
| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 프로필 영역 (아바타 + 이름 + ▼) 탭 | 멤버 선택 화면으로 이동 | 대시보드 상단 |
| 2 | 다른 멤버 카드 탭 | 멤버 ID 교체 저장 | 선택 화면 |
| 3 | — | 대시보드로 복귀 + 새 멤버 데이터 표시 | 탭 네비게이션 |

**위치:** 대시보드 상단, 날짜 표시 위 또는 옆
**스타일:** 아바타(작은 원) + 이름 + ▼ 드롭다운 아이콘

**구현 포인트:**
- 대시보드 상단에 `MemberProfile` 컴포넌트 배치
- `onPress` → `router.push('/select-member')` 또는 모달

### S-13-5. 멤버 변경 시 데이터 갱신

> **우선순위**: 중간 | **상태**: 🔴 미구현

멤버 변경 후 모든 화면의 데이터를 새 멤버 기준으로 갱신한다.

**갱신 대상:**
| 화면 | query | 파라미터 변경 |
|------|-------|------------|
| Dashboard | activeSession, members | memberId |
| History | calendar, dayDetail, monthlySummary | memberId |
| Members | (공통 데이터) | — |
| Admin | role 확인 → 탭 가시성 | memberId |

**갱신 전략:**
| 방법 | 장점 | 단점 |
|------|------|------|
| A. Apollo cache 초기화 (`client.resetStore()`) | 확실한 갱신 | 모든 캐시 날아감, 로딩 발생 |
| B. refetchQueries 지정 | 필요한 것만 갱신 | query 목록 관리 필요 |
| C. query variables 변경 (자동 refetch) | 자연스러움 | 이미 마운트된 화면만 갱신 |

**권장: C + A 조합**
- `selectedMemberId`가 변경되면 각 화면의 query variables가 자동 변경 → refetch
- 안전장치로 `client.resetStore()` 1회 호출

**구현 포인트:**
- 각 화면의 `useQuery`에서 `variables: { memberId: selectedMemberId }` 사용
- Zustand `selectedMemberId` 변경 → 자동으로 모든 query re-render
- Admin 탭 가시성: 새 멤버의 role에 따라 즉시 갱신
