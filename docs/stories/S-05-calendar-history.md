# S-05. 캘린더로 내 출석 기록 조회하기

> **탭**: History
> **관련 스펙**: REQ-1.1.1 ~ REQ-1.1.9 (History)
> **상태**: 🟡 부분 구현

---

## 유저 스토리

**AS** 스터디원
**I WANT TO** 캘린더에서 한눈에 내 출석 패턴을 확인
**SO THAT** 출석률과 지각 빈도를 시각적으로 파악할 수 있다

---

## 플로우

### 플로우 1: 캘린더 조회

```
[기록 탭 진입] → [이번 달 캘린더 표시, 오늘 날짜 선택]
→ [출석/지각/휴가 인디케이터 도트 확인]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 기록 탭 진입 | 현재 년/월 캘린더 렌더링 | 화면 상단 |
| 2 | — | 년/월 타이틀 표시 (e.g. "2026년 3월") | 캘린더 상단 헤더 |
| 3 | — | 오늘 날짜 강조 (원형 배경) + 자동 선택 | 캘린더 날짜 셀 |
| 4 | — | 각 날짜에 상태 인디케이터 도트 표시 | 날짜 숫자 아래 |

**잔디 블록 색상:**
| 상태 | 블록 색상 | 조건 |
|------|---------|------|
| 출석 (정상) | 🟢 초록 | 해당일 체크인 기록 있고, 지각 아님 |
| 지각 | 🟡 노랑 | 해당일 isLate == true |
| 휴가 | 🔵 파랑 | 해당일 DailyVacation 기록 있음 |
| 무단 결석 | 🔴 빨강 | 출석/휴가 기록 없는 평일 |
| 미래/주말/기록 없음 | ⬜ 빈 블록 | 기본 빈 셀 (회색 계열) |

### 플로우 2: 월 이동 (스와이프)

```
[캘린더에서 좌로 스와이프] → [다음 달 캘린더] → [해당 월 데이터 로딩]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 좌로 스와이프 | 다음 달 캘린더 표시 + 애니메이션 | 캘린더 영역 |
| 2 | — | `calendar` query 재호출 (새 year/month) | — |
| 3 | — | 해당 월의 인디케이터 도트 갱신 | 캘린더 날짜 셀 |
| 4 | — | 월간 요약 갱신 (S-07 연동) | 캘린더 하단 |
| 5 | 우로 스와이프 | 이전 달 캘린더 표시 | 캘린더 영역 |

### 플로우 3: 날짜 선택 → 일별 상세

```
[캘린더에서 3월 5일 탭] → [3월 5일 선택 하이라이트]
→ [하단에 3월 5일 출석 상세 표시] (→ S-06 연동)
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 특정 날짜 탭 | 해당 날짜 선택 하이라이트 | 캘린더 날짜 셀 |
| 2 | — | `dayDetail` query 호출 | — |
| 3 | — | 일별 상세 섹션 갱신 | 캘린더 아래 섹션 (S-06) |

## UI 레이아웃

### 잔디 스타일 캘린더

GitHub 잔디(contribution graph)처럼, 각 날짜를 사각형 블록으로 표현하고 상태에 따라 색상을 채운다.

```
┌─────────────────────────────────┐
│       ◀  2026년 3월  ▶          │  ← 월 헤더
│                                 │
│  일  월  화  수  목  금  토       │
│                                 │
│  ⬜  🟢  🟢  🟡  🟢  🟢  ⬜    │  ← 1주차 (주말=빈블록)
│  ⬜  🟢  🔵  🟢  🟢  🔴  ⬜    │  ← 2주차 (휴가=파랑, 무단=빨강)
│  ⬜  🟢  🟢  [🟢] ⬜  ⬜  ⬜    │  ← 3주차 ([오늘]=테두리 강조)
│  ⬜  ⬜  ⬜  ⬜  ⬜  ⬜  ⬜    │  ← 4주차 (미래=빈블록)
│                                 │
│  🟢 출석  🟡 지각  🔵 휴가  🔴 무단 │  ← 범례
│                                 │
├─────────────────────────────────┤
│  ▼ 일별 상세 (S-06)              │
├─────────────────────────────────┤
│  ▼ 월간 요약 (S-07)              │
└─────────────────────────────────┘
```

**블록 스타일:**
- 모서리 약간 둥근 사각형 (rounded-sm)
- 빈 블록: 연회색 배경 (`bg-gray-100`)
- 색상 블록: 각 상태별 색상 채움
- 오늘: 블록 테두리 강조 (검정/primary 보더)
- 선택된 날짜: 블록 아래 작은 인디케이터 또는 두꺼운 테두리

---

## 데이터 흐름

```
[화면 진입] → calendar(memberId, year, month)
    → [CalendarDay { date, status }] 배열 반환
    → 각 날짜 셀에 status 기반 도트 렌더링

[날짜 선택] → selectedDate 상태 변경
    → dayDetail(memberId, date) query 트리거

[월 이동] → year/month 상태 변경
    → calendar + monthlySummary query 재호출
```

## 서브 이슈

### S-05-1. 잔디 스타일 캘린더 그리드

> **우선순위**: 높음 | **상태**: 🔴 미구현

GitHub 잔디(contribution graph)처럼 각 날짜를 사각형 블록으로 렌더링하고, 상태에 따라 색상을 채운다.

**동작:**
- 7열(일~토) × 주 수 그리드 레이아웃
- 각 셀: 둥근 사각형 블록 (rounded-sm)
- 년/월 헤더 + 요일 라벨 행

**블록 색상 매핑:**
| 상태 | 색상 | Tailwind 예시 |
|------|------|-------------|
| 출석 (정상) | 초록 | `bg-green-400` |
| 지각 | 노랑 | `bg-yellow-400` |
| 휴가 | 파랑 | `bg-blue-400` |
| 무단 결석 | 빨강 | `bg-red-400` |
| 빈 블록 (미래/주말/기록 없음) | 연회색 | `bg-gray-100` |

**무단 결석 판정:**
```
평일(월~금) && 과거 날짜 && 출석 기록 없음 && 휴가 기록 없음 → 무단
```

**범례:** 캘린더 하단에 색상 범례 표시 (🟢 출석 🟡 지각 🔵 휴가 🔴 무단)

**구현 포인트:**
- FlatList 또는 View + flexWrap으로 7열 그리드
- `calendar` query의 `CalendarDay.status` → 색상 매핑 함수
- 서버: 무단 결석 상태 추가 필요 (기존 `NOT_ATTENDED`를 주말/미래 vs 무단으로 분리)

### S-05-2. 오늘 날짜 강조

> **우선순위**: 높음 | **상태**: 🔴 미구현

오늘 날짜 블록에 테두리 강조를 적용하고, 화면 진입 시 자동 선택한다.

**동작:**
| 시점 | 동작 |
|------|------|
| 화면 진입 | 오늘 날짜 자동 선택 + dayDetail 로딩 |
| 오늘 블록 스타일 | 다른 블록과 동일한 색상 채움 + 검정/primary 테두리 (2px) |

**UI:**
```
  🟢  🟢  [🟢]  ⬜  ⬜    ← [🟢] = 오늘 (테두리 강조)
```

**구현 포인트:**
- `isToday` 판정 → 조건부 `border-2 border-primary` 클래스
- `selectedDate` 초기값 = 오늘

---

### S-05-3. 상태별 블록 색상

> **우선순위**: 높음 | **상태**: 🔴 미구현

`calendar` query 결과의 status를 기반으로 각 블록에 적절한 색상을 적용한다.

**색상 규칙:**
| 상태 | 블록 색상 | 조건 |
|------|---------|------|
| 출석 | 🟢 초록 | 체크인 기록 있음, isLate == false |
| 지각 | 🟡 노랑 | 체크인 기록 있음, isLate == true |
| 휴가 | 🔵 파랑 | DailyVacation 기록 있음 |
| 무단 | 🔴 빨강 | 평일 + 과거 + 기록 없음 |
| 빈 블록 | ⬜ 연회색 | 미래 날짜, 주말, 해당 월 외 |

**복합 상태 처리 (반차 + 출석):**
- 우선순위: 지각 > 출석 > 휴가 (반차일 때 출석/지각 상태 우선 표시)
- 또는 블록을 반반 나누기 (상단 출석색 + 하단 휴가색) — 복잡도에 따라 결정

**서버 변경 필요:**
- `CalendarDay.status`에 `ABSENT` (무단) 상태 추가
- 평일 판정 로직 추가 (토/일 제외)

**구현 포인트:**
- `statusToColor` 매핑 유틸 함수
- NativeWind 동적 클래스 적용

### S-05-4. 스와이프 월 이동

> **우선순위**: 높음 | **상태**: 🔴 미구현

좌/우 스와이프로 다음/이전 달 캘린더로 이동한다.

**동작:**
| 제스처 | 결과 | 데이터 |
|-------|------|-------|
| 좌 스와이프 | 다음 달 캘린더 표시 | calendar(year, month+1) |
| 우 스와이프 | 이전 달 캘린더 표시 | calendar(year, month-1) |

**애니메이션:** 슬라이드 전환 (현재 달 → 밀려나감, 새 달 → 들어옴)

**구현 포인트:**
- `react-native-gesture-handler` + `Reanimated` 또는 `PagerView` 활용
- `year`, `month` 상태 관리 → 스와이프 시 변경
- 월 변경 시 `calendar` + `monthlySummary` query 재호출

### S-05-5. 날짜 선택

> **우선순위**: 높음 | **상태**: 🔴 미구현

날짜 블록을 탭하면 선택 상태로 표시하고, 일별 상세(S-06)를 로딩한다.

**동작:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|------------|-----------|
| 1 | 날짜 블록 탭 | 해당 블록 선택 하이라이트 (두꺼운 테두리 또는 확대) |
| 2 | — | `dayDetail(memberId, date)` query 트리거 |
| 3 | — | 캘린더 아래 일별 상세 섹션 갱신 (S-06) |
| 4 | 다른 날짜 탭 | 이전 선택 해제, 새 날짜 선택 |

**선택 스타일:**
- 선택된 블록: 살짝 확대 (scale 1.1) 또는 두꺼운 검정 테두리
- 오늘 + 선택: 오늘 테두리 + 선택 테두리 겹침

**구현 포인트:**
- `selectedDate` 상태 → 블록 onPress 핸들러
- 빈 블록(미래)도 탭 가능하되, 기록 없음 표시 (S-06 빈 상태)

### S-05-6. 월 헤더 화살표 이동

> **우선순위**: 중간 | **상태**: 🔴 미구현

스와이프 외에 ◀ ▶ 화살표 버튼으로도 월 이동이 가능하다.

**동작:**
| 버튼 | 결과 |
|------|------|
| ◀ (좌) | 이전 달 이동 |
| ▶ (우) | 다음 달 이동 |

**UI:**
```
│  ◀  2026년 3월  ▶  │
```

**구현 포인트:**
- 헤더에 좌/우 터치 영역 배치
- 스와이프(S-05-4)와 동일한 상태 변경 함수 공유

---

### S-05-7. 인접 월 데이터 프리페치

> **우선순위**: 낮음 | **상태**: 🔴 미구현

스와이프 시 빈 화면을 방지하기 위해 인접 월 데이터를 미리 로딩한다.

**동작:**
| 현재 월 | 프리페치 대상 |
|--------|-------------|
| 3월 표시 중 | 2월, 4월 calendar 데이터 미리 로드 |
| 월 이동 후 | 새로운 인접 월 프리페치 |

**구현 포인트:**
- Apollo `client.query` 또는 `useQuery` skip 옵션 활용
- 캐시에 데이터 저장 → 스와이프 시 즉시 렌더링
- 네트워크 비용 고려하여 calendar query만 프리페치 (monthlySummary는 on-demand)

---

### S-05-8. 빈 월 처리

> **우선순위**: 중간 | **상태**: 🔴 미구현

출석 기록이 전혀 없는 달에 빈 잔디 블록 캘린더를 표시한다.

**동작:**
| 상황 | 표시 |
|------|------|
| 기록 있는 달 | 색상 채워진 잔디 블록 |
| 기록 없는 달 | 전체 빈 블록(연회색) 그리드 유지 |

**UI:**
```
┌─────────────────────────────────┐
│       ◀  2025년 12월  ▶         │
│                                 │
│  ⬜  ⬜  ⬜  ⬜  ⬜  ⬜  ⬜    │
│  ⬜  ⬜  ⬜  ⬜  ⬜  ⬜  ⬜    │  ← 전부 빈 블록
│  ⬜  ⬜  ⬜  ⬜  ⬜  ⬜  ⬜    │
│  ⬜  ⬜  ⬜  ⬜  ⬜  ⬜  ⬜    │
│                                 │
│  이 달의 기록이 없습니다            │  ← 안내 문구 (회색)
└─────────────────────────────────┘
```

**구현 포인트:**
- 빈 달이어도 캘린더 그리드 구조 유지 (잔디 형태 일관성)
- 모든 블록 `bg-gray-100` + 하단 안내 텍스트
- 날짜 숫자는 표시하되, 블록 색상만 빈 상태
