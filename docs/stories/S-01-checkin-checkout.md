# S-01. 체크인/체크아웃으로 공부 시간 기록하기

> **탭**: Dashboard
> **관련 스펙**: REQ-1.3.1 ~ REQ-1.3.10, REQ-1.2.1 ~ REQ-1.2.2
> **상태**: 🟡 부분 구현

---

## 유저 스토리

**AS** 스터디원
**I WANT TO** 체크인/체크아웃 버튼으로 공부 시간을 기록
**SO THAT** 오늘 얼마나 공부했는지 추적할 수 있다

---

## 플로우

### 플로우 1: 첫 체크인

```
[대시보드 진입] → [출석 카드: "미출석" 상태] → [체크인 버튼 탭]
→ [상태 → "공부 중"] → [타이머 시작 HH:MM:SS]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 대시보드 탭 진입 | 오늘 날짜/요일 표시, "미출석" 상태 표시 | 출석 카드 (화면 상단) |
| 2 | "체크인" 버튼 탭 | `checkIn` mutation 호출 | 출석 카드 하단 버튼 영역 |
| 3 | — | 체크인 시간 기록, 상태 → "공부 중" | 출석 카드 상태 뱃지 |
| 4 | — | 타이머 시작 (HH:MM:SS, 1초 간격 갱신) | 출석 카드 중앙 타이머 영역 |
| 5 | — | 버튼 → "체크아웃"으로 변경 | 출석 카드 하단 버튼 영역 |

### 플로우 2: 체크아웃

```
[공부 중 상태] → [체크아웃 버튼 탭]
→ [타이머 정지] → [상태 → "완료"] → [오늘 총 공부 시간 표시]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | "체크아웃" 버튼 탭 | `checkOut` mutation 호출 | 출석 카드 하단 버튼 영역 |
| 2 | — | 체크아웃 시간 기록, 타이머 정지 | 출석 카드 중앙 타이머 영역 |
| 3 | — | 상태 → "완료" | 출석 카드 상태 뱃지 |
| 4 | — | 오늘 총 공부 시간 갱신 | 출석 카드 공부 시간 텍스트 |
| 5 | — | 버튼 → "다시 체크인"으로 변경 | 출석 카드 하단 버튼 영역 |

### 플로우 3: 다시 체크인 (복수 세션)

```
[완료 상태] → [다시 체크인 버튼 탭]
→ [새 세션 시작] → [타이머 0부터 재시작]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | "다시 체크인" 버튼 탭 | 새 `checkIn` mutation 호출 | 출석 카드 하단 버튼 영역 |
| 2 | — | 새 세션 생성, 타이머 00:00:00부터 시작 | 출석 카드 중앙 타이머 영역 |
| 3 | — | 총 공부 시간 = 이전 세션 + 현재 세션 누적 | 출석 카드 공부 시간 텍스트 |

## 에러 처리

| 상황 | 시스템 반응 | UI 위치 |
|------|-----------|---------|
| 체크인 요청 실패 | "체크인에 실패했습니다" 토스트 | 화면 하단 토스트 |
| 체크아웃 요청 실패 | "체크아웃에 실패했습니다" 토스트 | 화면 하단 토스트 |
| 네트워크 끊김 | 버튼 비활성화 + 재시도 안내 | 출석 카드 버튼 영역 |

---

## 데이터 흐름

```
[체크인 탭] → checkIn(memberId) → Session 생성 (checkOutTime: null)
    → Apollo cache 갱신 → UI 상태 전환

[체크아웃 탭] → checkOut(memberId) → Session 갱신 (checkOutTime 기록)
    → Apollo cache 갱신 → UI 상태 전환

[타이머] → activeSession(memberId) polling 30s
    → checkInTime 기준 경과 시간 계산 → 1초 간격 렌더링
```

## 서브 이슈

### S-01-1. 버튼 로딩 상태

> **우선순위**: 높음 | **상태**: 🔴 미구현

체크인/아웃 버튼 탭 후 mutation 응답이 올 때까지 로딩 상태를 표시하여 중복 탭을 방지한다.

**동작:**
| 시점 | 버튼 상태 | 설명 |
|------|---------|------|
| 탭 직후 | 스피너 표시 + 비활성화 | `loading: true` 동안 버튼 disabled |
| 응답 성공 | 다음 상태 버튼으로 전환 | 체크인 → 체크아웃 등 |
| 응답 실패 | 원래 버튼 복원 | 에러 토스트와 함께 (S-01-2) |

**구현 포인트:**
- Apollo `useMutation`의 `loading` 상태 활용
- 버튼 컴포넌트에 `disabled` + `ActivityIndicator` 조합

---

### S-01-2. 토스트 에러 메시지

> **우선순위**: 높음 | **상태**: 🔴 미구현

체크인/아웃 mutation 실패 시 화면 하단에 토스트 메시지를 표시한다.

**동작:**
| 실패 상황 | 토스트 메시지 | 자동 닫힘 |
|----------|------------|----------|
| 체크인 실패 | "체크인에 실패했습니다" | 3초 |
| 체크아웃 실패 | "체크아웃에 실패했습니다" | 3초 |

**UI 위치:** 화면 하단, 탭 바 위
**스타일:** 빨간 배경 + 흰색 텍스트 (에러 토스트)

**구현 포인트:**
- 토스트 라이브러리 도입 또는 커스텀 토스트 컴포넌트 (`shared/ui/toast`)
- `useMutation`의 `onError` 콜백에서 호출

---

### S-01-3. 네트워크 끊김 대응

> **우선순위**: 중간 | **상태**: 🔴 미구현

오프라인 상태에서 체크인/아웃 시도를 방지하고 안내 메시지를 표시한다.

**동작:**
| 네트워크 상태 | 버튼 | 안내 |
|-------------|------|------|
| 온라인 | 정상 활성화 | — |
| 오프라인 | 비활성화 (회색) | "네트워크 연결을 확인하세요" 배너 |
| 복귀 | 자동 활성화 | 배너 자동 숨김 |

**UI 위치:** 출석 카드 상단에 인라인 배너 (노란 배경)

**구현 포인트:**
- React Native `NetInfo` 또는 `@react-native-community/netinfo`
- 전역 네트워크 상태 → Zustand store 또는 Context

---

### S-01-4. 앱 백그라운드 복귀 시 타이머 보정

> **우선순위**: 높음 | **상태**: 🔴 미구현

앱이 백그라운드에서 돌아왔을 때 서버의 checkInTime 기준으로 타이머를 재계산한다.

**문제:** 백그라운드에서 setInterval이 멈추므로, 복귀 시 타이머가 실제 시간과 어긋남

**동작:**
| 시점 | 동작 |
|------|------|
| 앱 → 백그라운드 | 타이머 interval 유지 (OS가 자동 중단) |
| 앱 ← 포그라운드 복귀 | `activeSession` refetch → checkInTime 기준 경과 시간 재계산 |
| refetch 완료 | 타이머 값 보정, 1초 interval 재개 |

**구현 포인트:**
- React Native `AppState` 이벤트 리스너 (`active`, `background`)
- `appState === 'active'`일 때 `activeSession` refetch 트리거
- 또는 타이머를 매 tick마다 `checkInTime` 기준 `Date.now() - checkInTime`으로 계산 (drift 자체를 방지)

---

### S-01-5. 누적 공부 시간 표시

> **우선순위**: 중간 | **상태**: 🔴 미구현

현재 세션 타이머와 별도로, 오늘의 모든 세션을 합산한 총 공부 시간을 표시한다.

**동작:**
| 상태 | 현재 세션 타이머 | 누적 공부 시간 |
|------|---------------|-------------|
| 첫 세션 공부 중 | 01:30:00 (실시간) | 오늘 총: 1시간 30분 |
| 첫 세션 완료 | — (정지) | 오늘 총: 2시간 0분 |
| 두 번째 세션 공부 중 | 00:45:00 (실시간) | 오늘 총: 2시간 45분 |

**UI 위치:**
```
│      01:30:00              │  ← 현재 세션 타이머 (크게)
│  오늘 총 공부: 2시간 45분    │  ← 누적 시간 (작게, 서브 텍스트)
```

**구현 포인트:**
- 서버: `dayDetail(memberId, today)` → `totalDurationMinutes` 활용
- 또는 `activeSession` 응답에 `todayTotalMinutes` 필드 추가
- 공부 중일 때: 누적 = 완료된 세션 합 + 현재 세션 경과 시간

---

### S-01-6. "다시 체크인" 버튼 구분

> **우선순위**: 낮음 | **상태**: 🔴 미구현

첫 체크인과 재체크인(다시 체크인) 버튼을 시각적으로 구분한다.

**동작:**
| 상태 | 버튼 텍스트 | 스타일 |
|------|-----------|-------|
| 미출석 | "체크인" | Primary (메인 색상, 큰 버튼) |
| 공부 중 | "체크아웃" | Danger (빨간색 계열) |
| 완료 | "다시 체크인" | Secondary (아웃라인, 작은 버튼) |

**의도:** "다시 체크인"은 메인 액션이 아니라 보조 액션이므로, 첫 체크인보다 시각적 비중을 낮춘다.

**구현 포인트:**
- CheckButton 컴포넌트에 `variant` prop 추가 (primary / danger / secondary)
- 상태에 따라 variant 자동 전환

### S-01-7. 낙관적 업데이트 (Optimistic UI)

> **우선순위**: 중간 | **상태**: 🔴 미구현

mutation 응답을 기다리지 않고 UI를 먼저 전환하여 빠른 반응성을 제공한다.

**동작:**
| 시점 | UI 상태 | 데이터 |
|------|--------|-------|
| 체크인 탭 직후 | 즉시 "공부 중" + 타이머 시작 | 낙관적 Session 객체 |
| 서버 응답 성공 | 유지 | 실제 Session으로 교체 |
| 서버 응답 실패 | "미출석"으로 롤백 | 낙관적 데이터 제거 + 에러 토스트 |

**구현 포인트:**
- Apollo `useMutation`의 `optimisticResponse` 옵션 활용
- `update` 함수에서 cache 직접 수정
- 실패 시 Apollo가 자동으로 낙관적 데이터 롤백

```typescript
// 예시
checkIn({
  optimisticResponse: {
    checkIn: {
      __typename: 'Session',
      id: 'temp-id',
      checkInTime: new Date().toISOString(),
      checkOutTime: null,
      isLate: false,
    }
  }
})
```
