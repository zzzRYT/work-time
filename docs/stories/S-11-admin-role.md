# S-11. 관리자 역할 지정/해제하기

> **탭**: Admin (관리자 전용)
> **관련 스펙**: REQ-1.1.1 ~ REQ-2.3.1 (Admin)
> **상태**: 🔴 미구현

---

## 유저 스토리

**AS** 관리자
**I WANT TO** 다른 멤버에게 관리자 권한을 부여하거나 해제
**SO THAT** 여러 명이 스터디를 함께 관리할 수 있다

## 플로우

### 플로우 1: 관리 탭 접근

```
[관리자로 로그인] → [하단 탭 바에 "관리" 탭 표시] → [탭 진입]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | — | 선택된 멤버의 role == ADMIN 확인 | — |
| 2 | — | 하단 탭 바에 "관리" 탭 노출 | 탭 바 4번째 아이템 |
| 3 | "관리" 탭 탭 | 관리 화면 진입 | 전체 화면 |

**일반 멤버 (role == MEMBER):** 탭 바에 "관리" 탭 자체가 보이지 않음

### 플로우 2: 관리자 지정

```
[관리 화면] → [멤버 목록: 역할 표시] → ["지정" 버튼 탭]
→ [해당 멤버 ADMIN으로 변경] → [목록 즉시 갱신]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 관리 화면 진입 | 멤버 목록 + 현재 role 표시 | "관리자 지정" 섹션 |
| 2 | MEMBER 옆 "지정" 버튼 탭 | `updateMemberRole(memberId, ADMIN)` mutation | 멤버 행 우측 버튼 |
| 3 | — | role → ADMIN으로 변경, 버튼 → "해제"로 변경 | 멤버 행 |

### 플로우 3: 관리자 해제

```
[관리 화면] → [ADMIN 멤버 옆 "해제" 버튼 탭]
→ [해당 멤버 MEMBER로 변경]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | ADMIN 옆 "해제" 버튼 탭 | `updateMemberRole(memberId, MEMBER)` mutation | 멤버 행 우측 버튼 |
| 2 | — | role → MEMBER로 변경, 버튼 → "지정"으로 변경 | 멤버 행 |

## UI 레이아웃

### 탭 바 (관리자)

```
┌──────────┬──────────┬──────────┬──────────┐
│  대시보드  │   기록    │   멤버    │   관리   │  ← ADMIN만 노출
└──────────┴──────────┴──────────┴──────────┘
```

### 관리자 지정 섹션

```
┌─────────────────────────────────┐
│  관리자 지정                      │
│                                 │
│  ┌─────────────────────────┐    │
│  │ 🟣 홍길동   ADMIN  [해제] │    │  ← 현재 관리자
│  ├─────────────────────────┤    │
│  │ 🔵 김철수   MEMBER [지정] │    │  ← 일반 멤버
│  ├─────────────────────────┤    │
│  │ 🟢 이영희   MEMBER [지정] │    │
│  ├─────────────────────────┤    │
│  │ 🟠 박지민   ADMIN  [해제] │    │
│  └─────────────────────────┘    │
└─────────────────────────────────┘
```

**역할 뱃지:**
- ADMIN: 보라색/파란색 뱃지
- MEMBER: 회색 뱃지

**버튼:**
- "지정": 초록 아웃라인 버튼 → MEMBER를 ADMIN으로
- "해제": 빨간 아웃라인 버튼 → ADMIN을 MEMBER로

---

## 데이터 흐름

```
[화면 진입] → members query → 전체 멤버 + role 표시

[역할 변경] → updateMemberRole(memberId, role)
    → Member.role 업데이트
    → Apollo cache 자동 갱신 (__typename + id)
    → UI 즉시 반영

[탭 가시성] → 선택된 멤버의 role 확인
    → ADMIN → 탭 바에 "관리" 추가
    → MEMBER → 탭 바에서 "관리" 제거
```

## 서브 이슈

### S-11-1. 관리 탭 조건부 표시

> **우선순위**: 높음 | **상태**: 🔴 미구현

선택된 멤버의 role이 ADMIN일 때만 하단 탭 바에 "관리" 탭을 표시한다.

**동작:**
| 선택된 멤버 role | 탭 바 |
|------------|------|
| ADMIN | 대시보드 \| 기록 \| 멤버 \| **관리** (4탭) |
| MEMBER | 대시보드 \| 기록 \| 멤버 (3탭) |
| 미선택 | 대시보드 \| 기록 \| 멤버 (3탭) |

**멤버 변경 시:**
- ADMIN → MEMBER로 변경되면 "관리" 탭 즉시 숨김
- 관리 탭에 있던 중 숨겨지면 대시보드로 자동 이동

**구현 포인트:**
- Expo Router `(tabs)/_layout.tsx`에서 선택된 멤버의 role 기반 탭 조건부 렌더링
- `selectedMember.role === 'ADMIN'` → admin 탭 href 설정 / 아니면 null

---

### S-11-2. 멤버별 역할 목록 + 지정/해제

> **우선순위**: 높음 | **상태**: 🔴 미구현

전체 멤버의 현재 역할을 표시하고, 지정/해제 버튼으로 역할을 변경한다. 관리자는 여러 명이 될 수 있다.

**멤버 행 구조:**
```
│ 🟣 홍길동   [ADMIN 뱃지]  [해제] │  ← 관리자
│ 🔵 김철수   [MEMBER 뱃지] [지정] │  ← 일반 멤버
│ 🟢 이영희   [MEMBER 뱃지] [지정] │
│ 🟠 박지민   [ADMIN 뱃지]  [해제] │  ← 관리자 (복수 가능)
```

**역할 뱃지:**
| 역할 | 뱃지 스타일 |
|------|-----------|
| ADMIN | 보라색 배경 + 흰색 "관리자" 텍스트 |
| MEMBER | 회색 배경 + 흰색 "멤버" 텍스트 |

**버튼:**
| 현재 역할 | 버튼 | 스타일 | 결과 |
|---------|------|-------|------|
| MEMBER | "지정" | 초록 아웃라인 | → ADMIN |
| ADMIN | "해제" | 빨간 아웃라인 | → MEMBER |

**구현 포인트:**
- `members` query로 전체 멤버 + role 조회
- 버튼 탭 → `updateMemberRole(memberId, newRole)` mutation
- Apollo cache 자동 갱신 → UI 즉시 반영

---

### S-11-3. 마지막 관리자 해제 방지

> **우선순위**: 높음 | **상태**: 🔴 미구현

ADMIN이 1명뿐일 때 자기 자신을 해제하려 하면 경고를 표시하고 차단한다.

**동작:**
| 상황 | ADMIN 수 | "해제" 버튼 | 결과 |
|------|---------|-----------|------|
| 관리자 2명 이상 | 2+ | 활성 | 정상 해제 |
| 관리자 1명 (자기 자신) | 1 | 비활성 (회색) | 탭 시 경고 토스트 |

**경고 메시지:** "관리자가 최소 1명은 필요합니다"

**판정 로직:**
```
ADMIN 수 = members.filter(m => m.role === 'ADMIN').length
해제 대상이 ADMIN && ADMIN 수 === 1 → 차단
```

**구현 포인트:**
- 클라이언트: ADMIN 카운트 계산 → 1명이면 해당 행의 "해제" 버튼 disabled
- 서버: `updateMemberRole` resolver에서도 동일 검증 (최종 방어선)

### S-11-4. 역할 변경 성공/실패 토스트

> **우선순위**: 낮음 | **상태**: 🔴 미구현

역할 변경 mutation 완료/실패 시 토스트 메시지를 표시한다.

**토스트:**
| 상황 | 메시지 | 스타일 |
|------|-------|-------|
| 지정 성공 | "홍길동님을 관리자로 지정했습니다" | 성공 (초록) |
| 해제 성공 | "홍길동님의 관리자 권한을 해제했습니다" | 성공 (초록) |
| 변경 실패 | "역할 변경에 실패했습니다" | 에러 (빨강) |
| 마지막 관리자 | "관리자가 최소 1명은 필요합니다" | 경고 (주황) |

**구현 포인트:**
- `onCompleted` / `onError` 콜백에서 토스트 호출
- S-01-2 토스트 컴포넌트 재사용
