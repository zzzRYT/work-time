# S-12. 출근 시간/지각비 설정 변경하기

> **탭**: Admin (관리자 전용)
> **관련 스펙**: REQ-3.1.1 ~ REQ-4.4.1 (Admin)
> **상태**: 🔴 미구현

---

## 유저 스토리

**AS** 관리자
**I WANT TO** 스터디 출근 시간과 지각비 금액을 설정
**SO THAT** 기수별로 규칙을 유연하게 변경할 수 있다

## 플로우

### 플로우 1: 출근 시간 변경

```
[관리 탭] → [출근 시간 설정 섹션: 현재 "10:00"]
→ [시간/분 입력 → "저장" 탭] → [설정 갱신 → 지각 판정 기준 변경]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 관리 탭에서 스크롤 | "출근 시간 설정" 섹션 표시 | 관리자 지정 섹션 아래 |
| 2 | — | 현재 설정값 표시 (e.g. "10:00") | 섹션 상단 현재 값 |
| 3 | 시간(hour) 입력 필드 수정 | 숫자 키보드 표시, 0~23 제한 | 시간 입력 필드 |
| 4 | 분(minute) 입력 필드 수정 | 숫자 키보드 표시, 0~59 제한 | 분 입력 필드 |
| 5 | "저장" 버튼 탭 | `updateStudyStartTime(hour, minute)` mutation | 저장 버튼 |
| 6 | — | 현재 설정 표시 갱신 | 섹션 상단 현재 값 |
| 7 | — | 이후 체크인부터 새 시간 기준으로 지각 판정 | — |

### 플로우 2: 지각비 변경

```
[관리 탭] → [지각비 설정 섹션: 현재 "1,000원"]
→ [금액 입력 → "저장" 탭] → [설정 갱신 → 지각비 계산 기준 변경]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | — | "지각비 설정" 섹션 표시 | 출근 시간 설정 아래 |
| 2 | — | 현재 금액 표시 (e.g. "1,000원") | 섹션 상단 현재 값 |
| 3 | 금액 입력 필드 수정 | 숫자 키보드 표시, 0 이상 제한 | 금액 입력 필드 |
| 4 | "저장" 버튼 탭 | `updateLateFeeAmount(amount)` mutation | 저장 버튼 |
| 5 | — | 현재 설정 표시 갱신 | 섹션 상단 현재 값 |

### 플로우 3: 유효성 검증 실패

```

```

| 상황 | 에러 메시지 | UI 위치 |
|------|-----------|---------|
| 시간 0~23 범위 초과 | "0~23 사이 값을 입력하세요" | 시간 입력 필드 하단 |
| 분 0~59 범위 초과 | "0~59 사이 값을 입력하세요" | 분 입력 필드 하단 |
| 금액 음수 | "0 이상의 금액을 입력하세요" | 금액 입력 필드 하단 |

## 데이터 흐름

```
[화면 진입] → settings query
    → { studyStartHour, studyStartMinute, lateFeeAmount }
    → 입력 필드 초기값 설정

[출근 시간 저장] → updateStudyStartTime(hour, minute)
    → Settings 업데이트
    → 이후 checkIn resolver: settings에서 시간 읽어 지각 판정

[지각비 저장] → updateLateFeeAmount(amount)
    → Settings 업데이트
    → 이후 feeStatus / monthlySummary: settings에서 금액 읽어 계산

* 현재 상수로 하드코딩된 값 → Settings 모델로 이관 필요
```

## 서브 이슈

### S-12-1. 출근 시간 설정 UI

> **우선순위**: 높음 | **상태**: 🔴 미구현

현재 출근 시간을 표시하고, 시간/분 입력으로 변경할 수 있는 설정 섹션.

**구성:**
| 요소 | 설명 | UI |
|------|------|-----|
| 현재 값 | 현재 설정된 출근 시간 | "현재: 10:00" (읽기 전용) |
| 시간 입력 | 0~23 숫자 입력 | TextInput (numeric keyboard) |
| 분 입력 | 0~59 숫자 입력 | TextInput (numeric keyboard) |
| 저장 버튼 | mutation 호출 | Primary 버튼 |

**동작:**
| 단계 | 행동 | 결과 |
|------|------|------|
| 화면 진입 | `settings` query | 현재 값 표시 + 입력 필드 초기값 설정 |
| 값 수정 | 시간/분 변경 | 저장 버튼 활성화 (값 변경 감지) |
| 저장 탭 | `updateStudyStartTime(hour, minute)` | 현재 값 갱신 + 성공 토스트 |

**구현 포인트:**
- React Hook Form으로 폼 상태 관리
- `settings` query의 초기값으로 `defaultValues` 설정
- 저장 버튼: 값이 변경되지 않으면 비활성화

### S-12-2. 지각비 설정 UI

> **우선순위**: 높음 | **상태**: 🔴 미구현

현재 지각비 금액을 표시하고, 금액 입력으로 변경할 수 있는 설정 섹션.

**구성:**
| 요소 | 설명 | UI |
|------|------|-----|
| 현재 값 | 현재 설정된 지각비 | "현재: 1,000원" (읽기 전용) |
| 금액 입력 | 0 이상 숫자 입력 | TextInput (numeric keyboard) + "원" 접미사 |
| 저장 버튼 | mutation 호출 | Primary 버튼 |

**동작:**
| 단계 | 행동 | 결과 |
|------|------|------|
| 화면 진입 | `settings` query | 현재 금액 표시 + 입력 필드 초기값 |
| 값 수정 | 금액 변경 | 저장 버튼 활성화 |
| 저장 탭 | `updateLateFeeAmount(amount)` | 현재 값 갱신 + 성공 토스트 |

**구현 포인트:**
- 금액 입력: 숫자만 허용, 천 단위 콤마 자동 포맷 (표시용)
- 저장 시 콤마 제거 후 숫자로 전송

### S-12-3. 폼 유효성 검증

> **우선순위**: 높음 | **상태**: 🔴 미구현

입력값이 유효 범위를 벗어나면 인라인 에러 메시지를 표시하고 저장을 차단한다.

**검증 규칙:**
| 필드 | 규칙 | 에러 메시지 |
|------|------|-----------|
| 시간 (hour) | 0 ≤ hour ≤ 23, 정수 | "0~23 사이 값을 입력하세요" |
| 분 (minute) | 0 ≤ minute ≤ 59, 정수 | "0~59 사이 값을 입력하세요" |
| 금액 (amount) | amount ≥ 0, 정수 | "0 이상의 금액을 입력하세요" |
| 빈 값 | 필수 입력 | "값을 입력하세요" |

**에러 표시:**
```
│  ┌──────┐  :  ┌──────┐         │
│  │  25  │     │  00  │         │
│  └──────┘     └──────┘         │
│  🔴 0~23 사이 값을 입력하세요     │  ← 인라인 에러 (빨간 텍스트)
```

**구현 포인트:**
- Zod 스키마 정의:
  ```typescript
  const studyTimeSchema = z.object({
    hour: z.number().int().min(0).max(23),
    minute: z.number().int().min(0).max(59),
  })
  const lateFeeSchema = z.object({
    amount: z.number().int().min(0),
  })
  ```
- React Hook Form + `zodResolver` 연동
- 에러 시 저장 버튼 disabled + 인라인 에러 텍스트

### S-12-4. Settings 모델 Prisma 마이그레이션

> **우선순위**: 높음 | **상태**: 🔴 미구현

Settings 테이블을 DB에 생성하고 기본값을 seed한다.

**Prisma 스키마 (이미 정의됨):**
```prisma
model Settings {
  id               String   @id @default("default")
  studyStartHour   Int      @default(10)
  studyStartMinute Int      @default(0)
  lateFeeAmount    Int      @default(1000)
  updatedAt        DateTime @updatedAt
}
```

**작업:**
| 단계 | 명령 | 결과 |
|------|------|------|
| 1 | `npx prisma migrate dev` | Settings 테이블 생성 |
| 2 | seed.ts 업데이트 | 기본 Settings 레코드 삽입 (id: "default") |
| 3 | 검증 | `prisma studio`로 데이터 확인 |

**구현 포인트:**
- 싱글톤 패턴: id = "default"로 항상 1개 레코드만 존재
- `upsert`로 seed (이미 있으면 skip)

### S-12-5. 하드코딩 → DB 설정 전환

> **우선순위**: 높음 | **상태**: 🔴 미구현
> **연관**: S-02-2

서버의 하드코딩 상수를 Settings 테이블 조회로 전환한다.

**변경 대상:**
| 위치 | 현재 (하드코딩) | 변경 후 (DB) |
|------|-------------|------------|
| checkIn resolver | `STUDY_START_HOUR = 10` | `settings.studyStartHour` |
| checkIn resolver | `STUDY_START_MINUTE = 0` | `settings.studyStartMinute` |
| feeStatus resolver | `LATE_FEE_AMOUNT = 1000` | `settings.lateFeeAmount` |
| monthlySummary resolver | `LATE_FEE_AMOUNT = 1000` | `settings.lateFeeAmount` |
| constants.ts | 상수 파일 | 사용처 제거 또는 fallback으로만 유지 |

**settings service:**
```typescript
// src/services/settings.ts
async function getSettings(prisma: PrismaClient): Promise<Settings> {
  return prisma.settings.findFirstOrThrow({ where: { id: "default" } })
}
```

**구현 포인트:**
- 각 resolver에서 `getSettings(ctx.prisma)` 호출
- 매 요청마다 DB 조회 (설정 변경이 즉시 반영되어야 하므로)
- 성능 필요 시 추후 캐싱 고려

### S-12-6. 저장 성공/실패 피드백

> **우선순위**: 중간 | **상태**: 🔴 미구현

설정 저장 mutation 완료/실패 시 토스트 메시지를 표시한다.

**토스트:**
| 상황 | 메시지 | 스타일 |
|------|-------|-------|
| 출근 시간 저장 성공 | "출근 시간이 변경되었습니다" | 성공 (초록) |
| 지각비 저장 성공 | "지각비가 변경되었습니다" | 성공 (초록) |
| 저장 실패 | "설정 변경에 실패했습니다" | 에러 (빨강) |

**추가 동작:**
- 저장 성공 시 "현재" 값 표시 즉시 갱신
- 입력 필드는 새 저장값으로 유지

**구현 포인트:**
- `onCompleted` / `onError` 콜백에서 토스트 호출
- S-01-2 토스트 컴포넌트 재사용
