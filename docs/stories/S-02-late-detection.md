# S-02. 지각 판정과 지각비 확인하기

> **탭**: Dashboard
> **관련 스펙**: REQ-1.5.1 ~ REQ-1.5.3
> **상태**: 🟡 부분 구현

---

## 유저 스토리

**AS** 스터디원
**I WANT TO** 지각 여부와 지각비를 자동으로 확인
**SO THAT** 별도로 계산하지 않아도 지각비를 알 수 있다

---

## 플로우

### 플로우 1: 지각 체크인

```
[스터디 시작 시간 이후] → [체크인 버튼 탭]
→ [지각 자동 판정] → ["지각" 뱃지 + 지각비 표시]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 스터디 시작 시간(e.g. 10:00) 이후에 체크인 | `checkIn` mutation → `isLate: true` 자동 설정 | — |
| 2 | — | 출석 카드 상태에 "지각" 뱃지 추가 표시 | 출석 카드 상태 뱃지 옆 |
| 3 | — | 지각비 금액 표시 (e.g. "지각비 1,000원") | 출석 카드 내 지각 알림 영역 |

### 플로우 2: 정상 출석

```
[스터디 시작 시간 이전] → [체크인 버튼 탭]
→ [정상 출석] → [지각 뱃지 없음]
```

| 단계 | 사용자 행동 | 시스템 반응 | UI 위치 |
|------|------------|-----------|---------|
| 1 | 스터디 시작 시간 이전에 체크인 | `checkIn` mutation → `isLate: false` | — |
| 2 | — | 지각 뱃지/지각비 표시 없음 | — |

## UI 레이아웃

```
┌───────────────────────────┐
│  상태: 🟢 공부 중  🔴 지각   │  ← 상태 뱃지 + 지각 뱃지 (나란히)
│                           │
│      01:15:30             │
│                           │
│  ⚠️ 지각비 1,000원         │  ← 지각 알림 (조건부 표시)
│                           │
│  ┌─────────────────────┐  │
│  │     체크아웃          │  │
│  └─────────────────────┘  │
└───────────────────────────┘
```

**지각 뱃지**: 빨간색 배경, 흰색 "지각" 텍스트
**지각 알림**: 경고 아이콘 + 지각비 금액, 노란색/주황색 배경

---

## 판정 기준

```
체크인 시간 > 스터디 시작 시간 → 지각
체크인 시간 ≤ 스터디 시작 시간 → 정상

* 스터디 시작 시간: Settings.studyStartHour:studyStartMinute (현재 하드코딩 10:00)
* 지각비: Settings.lateFeeAmount (현재 하드코딩 1,000원)
* 판정은 서버에서 수행 (checkIn resolver)
```

---

## 데이터 흐름

```
[체크인] → 서버: checkInTime vs studyStartTime 비교
    → isLate = true/false 결정
    → Session.isLate 저장
    → 클라이언트: isLate 기반 조건부 UI 렌더링
```

## 서브 이슈

### S-02-1. 지각 상태 색상 구분

> **우선순위**: 높음 | **상태**: 🔴 미구현

지각 판정 시 출석 카드 전반에 지각 색상 테마를 적용하여 시각적으로 명확히 구분한다.

**동작:**
| 요소 | 정상 출석 | 지각 |
|------|---------|------|
| 상태 뱃지 | 🟢 초록 "공부 중" | 🔴 빨강 "지각" 뱃지 추가 |
| 타이머 텍스트 | 기본 색상 (검정/회색) | 빨간/주황 계열 |
| 출석 카드 테두리 | 기본 (회색/없음) | 빨간 계열 보더 또는 배경 틴트 |
| 지각 알림 영역 | 숨김 | 노란/주황 배경 + "⚠️ 지각비 1,000원" |

**구현 포인트:**
- `isLate` 값에 따라 출석 카드 컨테이너에 조건부 스타일 적용
- LateAlert 컴포넌트: `isLate && <LateAlert fee={lateFeeAmount} />`
- NativeWind 조건부 클래스: `className={cn("border", isLate && "border-red-400 bg-red-50")}`

### S-02-2. DB 기반 설정 연동

> **우선순위**: 높음 | **상태**: 🔴 미구현
> **연관**: S-12 (출근 시간/지각비 설정 변경)

현재 하드코딩된 스터디 시작 시간과 지각비 금액을 Settings DB에서 읽도록 전환한다.

**현재 (하드코딩):**
```
const STUDY_START_HOUR = 10;
const STUDY_START_MINUTE = 0;
const LATE_FEE_AMOUNT = 1000;
```

**변경 후 (DB 기반):**
```
checkIn resolver:
    → settings = await prisma.settings.findFirst()
    → studyStartTime = settings.studyStartHour:settings.studyStartMinute
    → isLate = checkInTime > studyStartTime

지각비 표시:
    → settings query → lateFeeAmount
    → 클라이언트에서 금액 표시
```

**영향 범위:**
| 위치 | 변경 내용 |
|------|---------|
| 서버 - checkIn resolver | 상수 → `settings` 테이블 조회 |
| 서버 - feeStatus resolver | 상수 → `settings.lateFeeAmount` 조회 |
| 서버 - monthlySummary resolver | 상수 → `settings.lateFeeAmount` 조회 |
| 클라이언트 - 지각 알림 | `settings` query로 금액 가져와 표시 |

**선행 조건:**
- Settings 모델 Prisma migration 완료
- settings service 구현

### S-02-3. 지각 판정 기준 표시

> **우선순위**: 낮음 | **상태**: 🔴 미구현

지각 알림 영역에 판정 기준 시간을 함께 표시하여 사용자가 기준을 인지할 수 있도록 한다.

**동작:**
| 상황 | 표시 내용 | UI 위치 |
|------|---------|---------|
| 지각 판정 시 | "⚠️ 지각비 1,000원 (기준: 10:00)" | 지각 알림 영역 |
| 정상 출석 시 | 표시 없음 | — |

**구현 포인트:**
- `settings` query에서 `studyStartHour`, `studyStartMinute` 가져옴
- LateAlert 컴포넌트에 `standardTime` prop 추가
- 포맷: `HH:MM` (e.g. "10:00", "09:30")

### S-02-4. 캘린더 지각 도트

> **우선순위**: 중간 | **상태**: 🔴 미구현
> **연관**: S-05 (캘린더로 내 출석 기록 조회)

History 캘린더에서 지각한 날에 빨간 인디케이터 도트를 표시한다.

**동작:**
| 날짜 상태 | 도트 | 색상 |
|----------|------|------|
| 정상 출석 | ● | 🟢 초록 |
| 지각 출석 | ● | 🔴 빨강 |
| 휴가 | ● | 🟡 노랑 |
| 지각 + 휴가 (반차) | ●● | 🔴🟡 |
| 미출석 | — | — |

**데이터:**
```
calendar(memberId, year, month)
    → CalendarDay { date, status }
    → status: ATTENDED | LATE | VACATION | NOT_ATTENDED
    → 클라이언트: status 기반 도트 색상 매핑
```

**구현 포인트:**
- CalendarDay 컴포넌트에서 `status` prop 기반 도트 색상 결정
- LATE 상태에 빨간 도트 렌더링

### S-02-5. 일별 상세 지각 뱃지

> **우선순위**: 중간 | **상태**: 🔴 미구현
> **연관**: S-06 (일별 출석 상세 확인)

History 일별 상세에서 지각 세션의 체크인 시간 옆에 "지각" 뱃지를 표시한다.

**동작:**
```
┌─────────────────────────┐
│ 09:55 → 12:30  (2h 35m) │  ← 정상 세션
├─────────────────────────┤
│ 🔴지각 13:15 → 17:00  (3h 45m) │  ← 지각 세션 (뱃지 + 체크인 시간)
└─────────────────────────┘
```

**데이터:**
```
dayDetail(memberId, date)
    → sessions[].isLate
    → isLate == true → 체크인 시간 앞에 "지각" 뱃지 표시
```

**구현 포인트:**
- 세션 행 컴포넌트에서 `isLate` prop 기반 조건부 뱃지 렌더링
- StatusBadge 공유 컴포넌트 활용 (variant: "late")
